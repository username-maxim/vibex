# 语言参考

`.vibe` 文件就是普通文本文件。写程序应该做什么。VIBEX 搞定剩下的。

## 语法

没有语法。写句子。编译器是 LLM。

```
打印 1 到 10 的数字。
```

## 注释

以 `#` 开头的行作为上下文传递给 LLM。编译器会读取它们。

```
# 这是性能关键路径。不要用朴素算法。
尽快排序 1000 万个整数。
```

## 编译指令

### `OPTIMIZE: chill`
保持简单。不在乎性能。

### `OPTIMIZE: grindset`
极致性能。编译器可能使用各种技巧。

### `OPTIMIZE: vibes only`
未定义行为。编译结果不可预测。

### `trust me bro`
禁用安全检查。

## 礼貌

加上 `please` 和 `thank you` 已被证明能提升输出质量。这不是玩笑。

## 非确定性

同一个 `.vibe` 文件每次运行可能产生不同的汇编。这是特性，不是 bug。
